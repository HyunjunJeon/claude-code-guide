# Checkpoint 핵심 동작

Checkpoint는 Claude Code가 파일 편집과 대화 흐름을 자동으로 추적해 이전 상태로 돌아갈 수 있게 하는 안전장치입니다. 별도로 저장 버튼을 누르는 기능이 아니라, 사용자 프롬프트와 Claude의 편집 흐름을 기준으로 자동 생성됩니다.

## 먼저 알아둘 것

| 항목 | 의미 |
|---|---|
| 자동 생성 | 모든 사용자 프롬프트가 새 checkpoint의 기준점이 됩니다 |
| 추적 대상 | Claude Code의 파일 편집 도구로 발생한 변경을 중심으로 추적합니다 |
| 접근 방법 | `Esc`를 두 번 누르거나 `/rewind`를 실행합니다 |
| 보존 기간 | 세션과 함께 보관되며 기본적으로 일정 기간 뒤 정리됩니다 |

## 언제 유용한가

- 큰 리팩터링을 여러 방향으로 실험할 때
- Claude가 잘못된 접근을 시작했지만 git commit 전이라 빠르게 되돌리고 싶을 때
- 코드만, 대화만, 또는 둘 다 되돌릴지 선택해야 할 때
- 실패한 시도에서 배운 내용을 요약으로 남기고 다음 방향을 잡고 싶을 때

## 주의할 점

Checkpoint는 git을 대체하지 않습니다. 빠른 탐색과 복구에는 checkpoint를 쓰고, 유지할 상태가 정해졌다면 git commit으로 고정합니다. 셸에서 직접 실행한 `rm`, `mv`, `cp` 같은 변경은 기대와 다르게 추적될 수 있으므로 파괴적인 명령 전에는 git 상태도 함께 확인합니다.
