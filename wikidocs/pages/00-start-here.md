이 섹션은 Claude Code를 처음 쓰는 독자가 전체 그림을 잡고 첫 작업까지 이어갈 수 있게 만든 입문 경로입니다. 설치와 실행, agentic coding loop, 기본 워크플로, 모범 사례를 먼저 훑은 뒤 필요한 기능 모듈로 이동하면 됩니다.

## 추천 읽기 순서

| 순서 | 문서 | 읽는 이유 |
|---|---|---|
| 1 | [Quickstart](https://wikidocs.net/345345) | 설치 후 첫 세션을 시작하는 최소 경로 |
| 2 | [Claude Code는 어떻게 동작하는가](https://wikidocs.net/345346) | Claude Code가 파일을 읽고, 수정하고, 검증하는 기본 루프 |
| 3 | [기능 개요](https://wikidocs.net/345347) | Memory, Skill, MCP, Hook, Plugin이 어디에 쓰이는지 큰 그림 |
| 4 | [공통 워크플로](https://wikidocs.net/345348) | 버그 수정, 테스트 작성, PR 준비 같은 실전 패턴 |
| 5 | [모범 사례](https://wikidocs.net/345349) | 좋은 지시, 검증, 컨텍스트 관리 습관 |

## 다음에 볼 섹션

- 터미널에서 명령을 어떻게 호출하는지 궁금하면 [CLI와 Slash Command](https://wikidocs.net/345350)를 읽습니다.
- 프로젝트 규칙을 Claude가 계속 기억하게 하려면 [Memory](https://wikidocs.net/345360)를 읽습니다.
- 반복 워크플로를 재사용하고 싶으면 [Skills](https://wikidocs.net/345381)를 읽습니다.
- 외부 도구와 데이터를 연결하려면 [MCP](https://wikidocs.net/345445)를 읽습니다.
