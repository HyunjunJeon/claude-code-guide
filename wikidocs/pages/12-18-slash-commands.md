슬래시 명령은 반복적인 워크플로를 이름 붙은 진입점으로 감싼 단축 명령입니다. 사용자가 긴 프롬프트를 기억하지 않아도 안정적인 호출 표면을 제공할 수 있습니다.

## 왜 중요한가

다음처럼 반복되는 흐름에 특히 잘 맞습니다.

- 코드 리뷰
- 배포 준비
- 프로젝트 전용 보고서
- 사고 대응
- 개발 환경 설정

실제로는 복잡한 로직 덩어리보다는, 잘 정의된 워크플로의 얇은 진입점으로 유지하는 편이 좋습니다.

## 설계 가이드

- 명령 이름은 짧고 기억하기 쉽게 둡니다.
- 필요한 입력과 부작용을 문서화합니다.
- 명령이 펼쳐지는 워크플로를 예측 가능하게 만듭니다.
- 조금씩만 다른 중복 명령을 남발하지 않습니다.

## 흔한 실수

- 기본 UX가 약한 문제를 슬래시 명령 남발로 덮는 것
- 거의 같은 명령을 여러 개 만드는 것
- 파괴적 동작을 무해해 보이는 이름 뒤에 숨기는 것
