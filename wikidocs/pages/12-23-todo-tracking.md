Todo tracking은 장시간 또는 다단계 작업에서 에이전트와 호스트가 공유할 수 있는 가벼운 작업 목록입니다.

## 무엇인가

자유 형식 설명만으로 진행 상황을 추측하게 두는 대신, 에이전트가 명시적인 작업 목록을 유지하고 상태를 갱신하며 무엇이 남았는지 보여줄 수 있게 합니다.

## 언제 유용한가

다음과 같은 경우에 특히 효과적입니다.

- 장시간 리팩터링
- 여러 파일을 건드는 마이그레이션
- 리뷰 워크플로
- 백그라운드 작업
- 운영자 대시보드

## 설계 가이드

좋은 todo 목록은 다음 특성을 가집니다.

- 짧다
- 결과 중심이다
- 작업 방향이 바뀌면 같이 갱신된다
- 호스트나 운영자에게 보인다

대화 전체를 또 하나의 목록으로 복제하는 수준이 되어서는 안 됩니다.

## 흔한 실수

- 실제 실행 상태와 목록이 어긋나는 것
- 체크 완료가 불가능할 만큼 모호한 항목을 쓰는 것
- 대화 전체를 todo 목록에 복사하는 것
