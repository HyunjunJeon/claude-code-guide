에이전트 루프는 Agent SDK 런타임의 중심입니다. 프롬프트를 받아 도구 결정, 도구 실행, 후속 추론, 최종 응답으로 이어지는 반복 사이클을 뜻합니다.

## 무엇인가

큰 흐름은 다음과 같습니다.

1. 앱이 프롬프트와 실행 옵션을 보냅니다.
2. Claude가 바로 답할지, 도구를 요청할지 결정합니다.
3. 호스트가 도구를 실행하고 결과를 다시 전달합니다.
4. Claude가 최종 답, 중단 조건, 턴 제한에 도달할 때까지 이 과정을 반복합니다.

권한과 훅은 이 루프 바깥에서 요청을 차단하거나 수정하거나 기록합니다.

## 언제 신경 써야 하나

다음 상황에서 루프 이해가 중요합니다.

- 에이전트가 왜 계속 도구를 호출하는지 디버깅할 때
- `maxTurns`로 과도한 반복을 막고 싶을 때
- 스트리밍 모드와 단일 실행 모드의 차이를 설명해야 할 때
- 정책과 로깅을 어느 경계에 넣어야 할지 결정할 때

## 멘탈 모델

SDK는 "프롬프트 입력, 답변 출력" 한 번으로 끝나는 래퍼가 아닙니다. 모델 단계와 도구 단계가 반복되는 이벤트 루프입니다. 그래서 세션, 훅, 권한, 관측성이 핵심이 됩니다.

## 제어 지점

- `query(...)`가 루프를 시작합니다.
- `maxTurns`가 반복 길이를 제한합니다.
- `allowedTools`, `disallowedTools`가 요청 가능한 도구를 좁힙니다.
- `permissionMode`, 선언형 설정, `canUseTool`이 승인 여부를 결정합니다.
- `PreToolUse`, `PostToolUse` 같은 훅이 실행 전후 동작을 관찰하거나 수정합니다.

## 흔한 실수

- 반복 도구 호출을 무조건 버그로 보는 것
- 차단된 도구 요청도 대화 흐름에는 영향을 준다는 점을 놓치는 것
- 승인이나 후속 입력이 필요한데도 단일 실행 모드를 택하는 것
- 세션이 루프 제어까지 대신한다고 생각하는 것
