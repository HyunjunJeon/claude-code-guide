# 독립형 vs Plugin 접근 방식

이 문서는 슬래시 커맨드를 단독(`/hello`)으로 만들 것인지, plugin으로 묶어 namespaced 명령(`/plugin-name:hello`)으로 배포할 것인지를 결정하는 기준을 설명합니다. 개인용 빠른 워크플로우와 팀 공유 사이에서 어느 방식이 적합한지 판단할 때 참고하세요. 일반적으로 단일 명령은 독립형, 여러 구성요소를 묶어 배포해야 하면 plugin이 적절합니다.

| 접근 방식 | 명령 이름 | 설정 | 적합한 용도 |
|----------|---------------|---|---|
| **독립형** | `/hello` | CLAUDE.md에서 수동 설정 | 개인용, 프로젝트별 |
| **Plugin** | `/plugin-name:hello` | plugin.json을 통한 자동화 | 공유, 배포, 팀 사용 |

빠른 개인 워크플로우에는 **독립형 slash command**를 사용하세요. 여러 기능을 번들로 묶거나, 팀과 공유하거나, 배포용으로 게시하려면 **plugin**을 사용하세요.
