# 문서 리팩터링

프로젝트 유형에 맞게 프로젝트 문서 구조를 리팩터링합니다:

1. **프로젝트 분석**: 유형(라이브러리/API/웹 앱/CLI/마이크로서비스), 아키텍처, 사용자 페르소나 식별
2. **문서 중앙화**: 기술 문서를 적절한 상호 참조와 함께 `docs/`로 이동
3. **루트 README.md**: 개요, 빠른 시작, 모듈/컴포넌트 요약, 라이선스, 연락처가 포함된 진입점으로 간소화
4. **컴포넌트 문서**: 설정 및 테스트 지침이 포함된 모듈/패키지/서비스 수준 README 파일 추가
5. **`docs/` 정리** 관련 카테고리별로:
   - 아키텍처, API 레퍼런스, 데이터베이스, 설계, 문제 해결, 배포, 기여 (프로젝트 필요에 맞게 조정)
6. **가이드 작성** (해당하는 항목 선택):
   - 사용자 가이드: 애플리케이션의 최종 사용자 문서
   - API 문서: API의 엔드포인트, 인증, 예제
   - 개발 가이드: 설정, 테스트, 기여 워크플로
   - 배포 가이드: 서비스/앱의 프로덕션 배포
7. **Mermaid 사용** 모든 다이어그램에 (아키텍처, 흐름, 스키마)

문서를 간결하고 스캔하기 쉽게, 프로젝트 유형에 맞게 작성합니다.
