# 서브에이전트

서브에이전트는 좁은 범위의 작업을 격리된 컨텍스트에서 처리한 뒤 결과를 부모 워크플로에 돌려주는 전문화된 하위 에이전트입니다.

## 왜 쓰는가

하나의 에이전트가 모든 세부사항을 한 컨텍스트에 들고 갈 필요가 없을 때 유용합니다. 책임 분리, 병렬 작업, 부모 에이전트의 조정 역할 유지에 도움이 됩니다.

좋은 예:

- 코드 탐색과 구현 분리
- 조사와 실행 분리
- 파일 범위별 소유권 분리
- 리뷰나 검증 전용 패스

## 멘탈 모델

부모는 조정자이고, 서브에이전트는 보조 작업자입니다. 중요한 경로를 무작정 떠넘기기보다는 다음 원칙을 지키는 편이 좋습니다.

- 과제를 좁게 정의한다
- 소유권을 명확히 준다
- 쓰기 범위가 겹치지 않게 한다
- 결과를 의도적으로 병합한다

## 흔한 실수

- 모호한 작업을 서브에이전트에 넘기는 것
- 당장 막힌 핵심 작업을 위임하고 기다리기만 하는 것
- 여러 에이전트를 같은 파일 집합에 보내는 것
- 병렬성의 이익만 보고 검토 비용을 무시하는 것
