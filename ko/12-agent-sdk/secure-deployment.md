# 보안 배포

보안 배포는 운영 환경에서 SDK 기반 에이전트가 무엇에 접근하고, 무엇을 바꾸고, 무엇을 유출할 수 있는지를 최소화하는 작업입니다.

## 핵심 통제 지점

최소한 다음은 명확해야 합니다.

- 최소 권한 자격 증명
- 좁은 도구 허용 목록
- 분명한 권한 정책
- 프롬프트 밖에서의 비밀 관리
- 네트워크 경계와 감사 로그

## 실전 가이드

- 기본값은 읽기 전용 또는 매우 좁은 쓰기 범위로 둡니다.
- 운영 비밀은 저장소 파일이 아니라 호스팅 계층에서 주입합니다.
- 테넌트와 작업 공간을 격리합니다.
- 파괴적 작업은 명시적이고 검토 가능하게 만듭니다.
- 비정상 동작을 관측성과 경보에 연결합니다.

## 흔한 실수

- 공유 환경이나 민감한 환경에서 `bypassPermissions`를 쓰는 것
- 워크플로에 필요 이상으로 넓은 셸 접근을 여는 것
- 프롬프트, 출력, 도구 페이로드를 마스킹 계획 없이 로그에 남기는 것
